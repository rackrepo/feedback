# RackRepo Feedback

This repo exists primarily to make it easy to leave feedback on
[RackRepo](https://rackrepo.com), the mini-rack planner.

The app's source lives elsewhere and isn't public. 

## What this repo is for

- **Reporting a bug** you hit in the planner.
- **Requesting a feature** you wish existed.
- **Asking a question** when the answer might help other homelabbers too.

## What this repo isn't

- Not the source code. RackRepo is closed-source.
- Not a support channel with an SLA. One person reads these. Replies
  are best-effort.

## How to file feedback

1. **Search existing issues** first. Someone may already have hit the
   same thing, and piling on an existing issue is more useful than a
   fresh one.
2. **Open a new issue** using one of the templates:
   - [Bug report](../../issues/new?template=bug_report_basic.yml) for
     something that's broken or behaving oddly.
   - [Feature request](../../issues/new?template=feature_request_basic.yml)
     for something you wish the planner did.
3. **Fill in the fields.** The templates ask for the stuff that's
   usually needed to act on a report. Skipping optional fields is
   fine if they don't apply.

## Before you file

- The planner is a browser app. If something looks wrong, a quick
  reload sometimes clears it, and the browser console (F12) often has
  a useful error. Pasting that into the bug report helps a lot.
- Plans live in your browser's local storage, not on a server. If
  something corrupts a plan, exporting the JSON (there's a button in
  the app) and attaching it to the issue makes the bug reproducible.
- Mobile and desktop are both in scope. "Works on desktop but breaks
  on my phone" is a valid report.
